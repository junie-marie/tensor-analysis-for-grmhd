# Athena++ Output Documentation: 2D RMHD Blast Wave

This document details the data output generated by Athena++. The simulation produces a sequence of HDF5 files containing the **conserved variables** for a 2D Relativistic Magnetohydrodynamics (RMHD) blast wave.

---

### **1. Output File Structure**

Athena++ consolidates the data from all processors into single, combined `.athdf` files per output step. 

* **Data Files:** `blast.out1.00000.athdf` through `blast.out1.00010.athdf`. 
    * *Note:* The simulation runs to `tlim = 1.0` with outputs every `dt = 0.1`, resulting in 11 total data dumps (including the initial state at $t = 0$).
* **Visualization Metadata:** `blast.out1.00000.athdf.xdmf` (and subsequent `.xdmf` files). 
    * *Note:* These eXtensible Data Model and Format (XDMF) files are lightweight XML wrappers that allow visualization software (like ParaView or VisIt) to properly read the underlying `.athdf` files.

---

### **2. Mesh & Domain Properties**

The data inside the HDF5 files is structured around the MeshBlocks defined in your input file.

* **Global Domain:** 200 x 200 x 1 zones.
* **Physical Bounding Box:** X1: [-6.0, 6.0], X2: [-6.0, 6.0], X3: [-0.5, 0.5].
* **MeshBlock Decomposition:** The data is stored in **4 MeshBlocks**, each containing 100 x 100 x 1 zones. This naturally maps to your 4 MPI processes (`mpirun -np 4`).

---

### **3. Variable Dictionary (`variable = cons`)**

Because the output is set to `cons` under an RMHD build, the data arrays in the HDF5 file represent the **Eulerian conserved variables** rather than primitive variables (like rest-mass density or proper velocity). 

The following arrays are contained within each `.athdf` file. Note that $\Gamma$ represents the Lorentz factor, $\rho$ is the rest-mass density, $w$ is the proper enthalpy, $\mathbf{v}$ is the fluid velocity, $\mathbf{B}$ is the magnetic field, and $p$ is the gas pressure.

| HDF5 Dataset Name | Dimensions | Description | Mathematical Definition (SRMHD) |
| :--- | :--- | :--- | :--- |
| `dens` | `[1, nx3, nx2, nx1]` | Conserved rest-mass density in the lab frame. | $D = \Gamma \rho$ |
| `mom` | `[3, nx3, nx2, nx1]` | Conserved momentum density vector ($M_x, M_y, M_z$). | $\mathbf{M} = w \Gamma^2 \mathbf{v} + \mathbf{E} \times \mathbf{B}$ |
| `Etot` | `[1, nx3, nx2, nx1]` | Total energy density in the lab frame (excluding rest mass, depending on exact Athena++ config, but typically total $E$). | $E = w \Gamma^2 - p + \frac{1}{2}(B^2 + E^2)$ |
| `Bcc` | `[3, nx3, nx2, nx1]` | Cell-centered magnetic field vector ($B_x, B_y, B_z$). | $\mathbf{B}$ |

*Note: In Athena++, 3-vector datasets like `mom` and `Bcc` store their components in the first dimension (e.g., index 0 = X1, index 1 = X2, index 2 = X3).*

---

### **4. Reading the Data**

Because Athena++ HDF5 files utilize a specific MeshBlock structure, standard HDF5 readers (like `h5py` in Python) require manual stitching of the blocks. 

To easily read and analyze this data in Python, you should use the official script provided in the Athena++ repository:
`athena/vis/python/athena_read.py`

**Basic Usage Example (Python):**
```python
import sys
sys.path.insert(0, '/path/to/athena/vis/python')
import athena_read

# Read the 10th output file (t = 1.0)
data = athena_read.athdf('blast.out1.00010.athdf')

# Extract X and Y coordinates, and conserved density
x = data['x1f']
y = data['x2f']
density = data['dens'][0] # [0] selects the single Z-plane
```
